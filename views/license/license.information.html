Source Code Contributions

All source code contributions to The Appverse Project must be made under the terms of the Appverse Public License Version 2.0. If a source code submission is subject to multiple licenses, those licenses must also be compatible with the terms of the Appverse Public License Version 2.0. Please refer to the following document for more information:

  • Appverse Public License Version 2.0

NOTE: Appverse Public License is based ont the Mozilla Public License 2.0 and contains slight changes to the original to comply with Spanish law.

Please note that The Appverse Project cannot accept source code contributions from individuals who have not submitted an Individual Contributor License Agreement (ICLA) or from corporations that have not submitted a Corporate Contributor License Agreement (CCLA). These agreement documents are designed to increase transparency, ensure license awareness and protect you and the Appverse Project from inadvertent intellectual property rights infringements. The Appverse Project reserves the right to refuse contributor applications to the project’s master repositories at their discretion.

The following sections explain the requirements in more detail.

Individual Contributors

If you’re an individual developer and would like to become a committer to the Appverse project, we will require a completed and signed Individual Contributor License Agreement (ICLA). You can find the document here:

  • Appverse Individual Contributor License Agreement Version 1.1 (ICLA)

The completed document should be sent to Appverse as described within the ICLA document itself.

The Appverse Project welcomes pull requests from developers providing patches to existing code without an ICLA. If you wish to contribute additional original work to the project, we will require an ICLA.

If you’re contributing to the project on behalf of your employer, specifically, if your employer is sponsoring your participation in the project and owns the intellectual property of the work, we will also need a Corporate Contributor License Agreement (CCLA) to ensure the contributions are fully disclosed and authorised.

The following sections explain the requirements in more detail.

  • Appverse Corporate Contributor License Agreement Version 1.1 (CCLA)

The completed document should be sent to Appverse as described within the CCLA document itself. Additionally, for each of the nominated employees we will also need a completed and signed Individual Contributor License Agreement (ICLA). You can find the document here:

  • Appverse Individual Contributor License Agreement Version 1.1 (ICLA)

The completed document should be sent to Appverse as described within the ICLA document itself.

The Appverse Project welcomes pull requests from developers providing patches to existing code without a CCLA and ICLA on file. If you wish to become an official committer to the project, especially when providing new code that may be subject to other copyrights/intellectual property claims, we will require an authorised CCLA and ICLA.
